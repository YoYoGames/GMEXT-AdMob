Workflow

This is the recommended workflow for using AdMob extension functions calls both on Android and iOS.

iOS

  • Import the AppTrackingTransparency extension from marketplace. This is required for personalized ads.
  • Request app tracking in an initialization room before initializing AdMob.

Android & iOS

  • Set desired configurations: AdMob_SetTestDeviceId, AdMob_Targeting_COPPA, AdMob_Targeting_UnderAge and AdMob_Targeting_MaxAdContentRating.
  • Initialize the extension: AdMob_Initialize
  • Wait for callback (success/failure)
  • Handle failure (don’t continue any further)
  • Request for consent information update: AdMob_Consent_RequestInfoUpdate
  • Check status using AdMob_Consent_GetStatus
  • Proceed to loading and showing AdMob_Consent_Load / AdMob_Consent_Show
  • Now you can finally init and load your ads using the function pairs:
  • ~~AdMob_Banner_Init~~ / AdMob_Banner_Create
  • ~~AdMob_Interstitial_Init~~ / AdMob_Interstitial_Load
  • ~~AdMob_RewardedVideo_Init~~ / AdMob_RewardedVideo_Load
  • ~~AdMob_RewardedInterstitial_Init~~ / AdMob_RewardedInterstitial_Load
  • After loading is successful you can show your ads with the corresponding show function.
